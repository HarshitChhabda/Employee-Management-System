# HRMS Pro Max - अपडेट फीचर गाइड

## 1. अपडेट फीचर का परिचय

यह फीचर एप्लिकेशन को अपडेट करने की सुविधा देता है। अपडेट **दो तरीकों** से सिस्टम में आ सकता है:

| तरीका | स्रोत | विवरण |
|-------|-------|--------|
| **पेनड्राइव / लोकल** | `pendrive` | पेनड्राइव में `.hrms-update.json` और `update_package.json` डालकर स्कैन करें |
| **सुपरबेस (क्लाउड)** | `supabase` | सुपरबेस डेटाबेस से अपडेट सिंक करें (Sync Supabase बटन) |

---

## 2. तीनों यूज़र रोल्स में व्यवहार

### (क) ब्रांच एडमिन (Branch Admin / Base User) - `ROLE_BRANCH`
| एक्शन | अनुमति? |
|-------|---------|
| अपडेट लिस्ट देखना | ✅ हाँ |
| अपडेट जांचना (Check Updates) | ✅ हाँ |
| पेनड्राइव स्कैन करना | ✅ हाँ |
| सुपरबेस से सिंक करना | ✅ हाँ |
| अपडेट इंस्टॉल करना | ✅ हाँ |

### (ख) एचओ एडमिन (HO Admin / Admin User) - `ROLE_HO`
| एक्शन | अनुमति? |
|-------|---------|
| अपडेट लिस्ट देखना | ✅ हाँ |
| अपडेट जांचना (Check Updates) | ✅ हाँ |
| पेनड्राइव स्कैन करना | ✅ हाँ |
| सुपरबेस से सिंक करना | ✅ हाँ |
| अपडेट इंस्टॉल करना | ✅ हाँ |

### (ग) सुपर एडमिन (Super Admin / Super User) - `ROLE_SUPER`
| एक्शन | अनुमति? |
|-------|---------|
| अपडेट लिस्ट देखना | ✅ हाँ (केवल देख सकता है) |
| अपडेट जांचना (Check Updates) | ❌ नहीं (बटन दिखेगा नहीं) |
| पेनड्राइव स्कैन करना | ❌ नहीं (सेक्शन दिखेगा नहीं) |
| सुपरबेस से सिंक करना | ❌ नहीं (बटन दिखेगा नहीं) |
| अपडेट इंस्टॉल करना | ❌ नहीं (इंस्टॉल बटन दिखेगा नहीं) |

> **महत्वपूर्ण:** Super Admin केवल **Read-Only** है। वह अपडेट को केवल देख सकता है, अपलोड या इंस्टॉल नहीं कर सकता।

## 3. अपडेट अपलोड कैसे करें (Branch Admin / HO Admin के लिए)

### विधि 1: पेनड्राइव के माध्यम से

**स्टेप 1:** पेनड्राइव में निम्न फ़ाइलें बनाएं:

```
D:\
├── .hrms-update.json        # अपडेट मैनिफेस्ट (जरूरी)
├── update_package.json      # फाइल मैपिंग (जरूरी)
├── dist/                     # अपडेट फाइलें
│   ├── app.js
│   └── ...
└── assets/
    └── style.css
```

**.hrms-update.json** का फॉर्मेट:
```json
{
  "version": "1.2.0",
  "title": "बग फिक्स रिलीज़",
  "description": "अटेंडेंस कैलकुलेशन में सुधार",
  "file_size": 2048000,
  "checksum": "sha256-xxx",
  "module_scope": "all"
}
```

**update_package.json** का फॉर्मेट:
```json
{
  "files": [
    { "source": "dist/app.js", "dest": "app.js" },
    { "source": "assets/style.css", "dest": "assets/style.css" }
  ]
}
```

**स्टेप 2:** एप्लिकेशन में Settings > App Updates पर जाएं।

**स्टेप 3:** "Pendrive / Local Scan" सेक्शन में ड्राइव पथ दर्ज करें (जैसे `D:\` या `E:\hrms-update`)।

**स्टेप 4:** **Scan** बटन पर क्लिक करें। सिस्टम पेनड्राइव में `.hrms-update.json` ढूंढेगा और अपने आप रजिस्टर कर देगा।

**स्टेप 5:** अपडेट लिस्ट में नया अपडेट "available" स्टेटस के साथ दिखने लगेगा।

---

### विधि 2: सुपरबेस (क्लाउड) के माध्यम से

**स्टेप 1:** सुपरबेस डैशबोर्ड पर जाएं और `app_updates` टेबल में नया रिकॉर्ड डालें:
```sql
INSERT INTO app_updates (id, version, title, description, status, file_url, file_size, module_scope)
VALUES (gen_random_uuid(), '1.2.0', 'बग फिक्स रिलीज़', 'अटेंडेंस कैलकुलेशन में सुधार', 'available', 'updates/v1.2.0.zip', 2048000, 'all');
```

**स्टेप 2:** अपडेट फाइल को Supabase Storage के `app-updates` बकेट में अपलोड करें।

**स्टेप 3:** एप्लिकेशन में Settings > App Updates > **Sync Supabase** बटन पर क्लिक करें।

**स्टेप 4:** सभी उपलब्ध अपडेट लोकल डेटाबेस में कॉपी हो जाएंगे।

---

## 4. अपडेट डाउनलोड / इंस्टॉल कैसे करें

### ब्रांच एडमिन और एचओ एडमिन के लिए:

**स्टेप 1:** Settings > App Updates पर जाएं।

**स्टेप 2:** **Check Updates** बटन पर क्लिक करके देखें कि कोई अपडेट उपलब्ध है या नहीं।

**स्टेप 3:** अगर अपडेट उपलब्ध है, तो उसके आगे **Install** बटन (हरा) दिखेगा। उस पर क्लिक करें।

**स्टेप 4:** सिस्टम अपडेट को इंस्टॉल करेगा और स्टेटस "installed" हो जाएगा।

**इंस्टॉल प्रक्रिया:**
- पेनड्राइव अपडेट: `file_path` से फाइलें कॉपी होकर एप्लिकेशन डायरेक्टरी में लग जाएंगी
- सुपरबेस अपडेट: पहले सुपरबेस स्टोरेज से फाइल डाउनलोड होगी, फिर इंस्टॉल होगी

### सुपर एडमिन के लिए:

सुपर एडमिन केवल Settings > App Updates पर जाकर यह देख सकता है कि कौन-कौन से अपडेट उपलब्ध हैं या इंस्टॉल हो चुके हैं। वह **न तो अपडेट अपलोड कर सकता है और न ही इंस्टॉल** कर सकता है।

## 5. ऑटोमैटिक अपडेट नोटिफिकेशन

- साइडबार (Sidebar) हर **60 सेकंड** में अपने आप चेक करता है कि कोई नया अपडेट उपलब्ध है या नहीं
- अगर अपडेट उपलब्ध है, तो Settings आइकन पर एक **नोटिफिकेशन बैज (डॉट)** दिखने लगता है
- यह तीनों यूज़र्स (Branch, HO, Super) के लिए काम करता है

## 7. महत्वपूर्ण नोट्स

1. **Super Admin (ROLE_SUPER)** के लिए पूरा Updates Tab **Read-Only** मोड में होता है। यह `isReadOnly = isSuperAdmin` वेरिएबल से कंट्रोल होता है (`MasterSettings.tsx:30`)।

2. पेनड्राइव स्कैन करते समय, सिस्टम केवल `.hrms-update.json` नाम की फ़ाइल को पहचानता है। फ़ाइल का नाम बिल्कुल सही होना चाहिए।

3. अपडेट **version** यूनिक होना चाहिए। अगर वही वर्जन पहले से मौजूद है, तो रजिस्टर करने पर एरर आएगी।

4. सुपरबेस सिंक के लिए `system_config` टेबल में `supabase_url` और `supabase_anon_key` कॉन्फ़िगर होना जरूरी है।
